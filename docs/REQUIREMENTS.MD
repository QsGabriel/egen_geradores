FUNCIONALIDADE 9. Requisitos Não Funcionais 

**Nota 1 - Segurança da Informação** 

* RNF 01 - Senhas devem ser armazenadas utilizando hash seguro 


* RNF 02 - Controle de acesso baseado em perfis e permissões 


* RNF 03 - Sistema web desenvolvido em React (frontend) e Node.js/Python (backend) 


* RNF 04 - Todo o tráfego de dados deve ocorrer via HTTPS 



**Nota 2 - Desempenho e Escalabilidade** 

* RNF 05 - O sistema deve suportar crescimento gradual de usuários sem reescrita de código 


* RNF 06 - Operações críticas (cadastros, consultas) devem responder em até 2 segundos (sem considerar rede) 


* RNF 07 - Processos mais pesados (ex.: geração de PDF) devem ser assíncronos quando possível 


* ANF 08 - Arquitetura preparada para escalar horizontalmente no backend 



**Nota 3 - Arquitetura e Qualidade de Código** 

* RNF 08 - Arquitetura baseada em camadas (frontend, backend, banco) 


* RNF 09 - Backend estruturado em API REST 


* RNF 10 - Código modular e reutilizável 



**Nota 4 - Usabilidade e Experiência do Usuário (UX)** 

* RNF 11 - Interface intuitiva, voltada para usuários não técnicos 


* RNF 12 - Feedback visual para ações do usuário (sucesso, erro, carregamento) 


* RNF 13 - Interface responsiva para desktop e tablet 


* RNF 14 - Layout consistente em todo o sistema (Personalizado para a EGEN)
